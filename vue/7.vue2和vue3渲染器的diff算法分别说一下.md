"Vue 2 和 Vue 3 的 diff 算法核心都是同层比较，但实现策略和优化点有很大不同。"
#### 1. Vue 2 的 Diff 算法（双端比较）
**核心思想**：采用"双端比较"策略，通过四个指针在头尾同时进行比较。  
**具体过程**：  
1. 先比较新旧节点的头头、尾尾
2. 再交叉比较头尾、尾头
3. 如果都不匹配，就用 key 在旧节点中查找
4. 最后处理剩余节点
**特点**
- 时间复杂度 O(n)，能处理大部分场景
- 但某些情况会做不必要的移动
- 没有预处理，从头开始比较
#### 2.Vue 3 的 Diff 算法（快速 Diff）
**核心思想**：三段式处理 + 最长递增子序列优化。
**具体过程**：
1. **预处理阶段**：先跳过相同的前缀和后缀
2. **中间部分处理**：
- 为新节点建立key到索引的映射
- 遍历旧节点，用key查找可复用节点
3. **移动优化**：用&zwnj;**最长递增子序列(LIS)**&zwnj;确定哪些节点保持不动，只移动必要节点
#### 3.主要区别对比
| 方面 | Vue2 | Vue3 |
|:-|:-|:-|
|**算法名称**|双端比较|快速Diff|
|**预处理**|无|有前后缀跳过|
|**核心优化**|双端四指针|最长递增子序列|
|**移动效率**|可能多余移动|最小化移动|
|**使用场景**|通用|更适合顺序变动小的列表|
#### 4.为何这样设计？
Vue 3 的改进主要是为了：
1. **实际场景优化**：大部分列表操作是头尾增减，预处理能快速跳过
2. **移动最小化**：最长递增子序列确保只移动必要节点
3. **结合编译优化**：与 Patch Flag 静态标记配合，减少 diff 范围
